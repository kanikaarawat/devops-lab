## Commands Studied on Day 1:  

### 1. git init  
Initializes a new Git repository in the current directory.

### 2. git add  
Stages changes to be committed to the repository.

### 3. git commit  
Commits staged changes with a descriptive message.

### 4. git status  
Displays the current status of files in the working directory and staging area.

### 5. git remote add origin  
Connects the local repository to a remote repository.

### 6. git push  
Pushes committed changes from the local repository to the remote repository.

### 7. git mv  
Renames or moves files within the repository and stages the operation automatically.

### 8. nano  
The `nano` text editor was used to create and edit the `Readme.txt` file locally before committing the changes to the Git repository.
